1"""Utilities for testing the dataset fetchers 

2 

3Unit tests should not depend on an internet connection nor on external 

4resources such as the servers from which we download datasets. Otherwise, tests 

5can fail spuriously when a web service is unavailable, and tests are slow 

6because downloading data takes a lot of time. 

7 

8Therefore in the tests, we fake the downloads: the function from the requests 

9library that would normally download a file is replaced ("patched") by a 

10"mock", a function that mimicks its interface but doesn't download anything and 

11returns fake data instead. 

12 

13As we only patch functions from urllib and requests, nilearn code is unaware of 

14the mocking and can be tested as usual, as long as we provide fake responses 

15that look similar to those we would obtain from dataset providers if we 

16actually sent requests over the network. 

17 

18This module provides the utilities for setting up this mocking: patching the 

19relevant requests and urllib functions, and creating the fake responses. The 

20function from the requests library that nilearn uses to send requests is 

21patched (replaced) by a `Sender` object defined in this module. The 

22corresponding docstring details how individual tests can configure what fake 

23responses it should return for specific URLs. 

24 

25To make sure tests don't rely on previously existing data and don't write 

26outside of temporary directories, this module also adds fixtures to patch the 

27home directory and other default nilearn data directories. 

28 

29"""

143class Sender: 

144 r"""Mock class used to patch requests.sessions.Session.send 

145 

146 In nilearn's tests this replaces the function used by requests to send 

147 requests over the network. 

148 

149 Test functions can configure this object to specify what response is 

150 expected when sending requests to specific URLs. This is done by adding 

151 items to the ordered dictionary self.url_mapping 

152 

153 When a Sender receives a request, it tries to match it against the keys in 

154 url_mapping, and then against urls found in 

155 nilearn/datasets/tests/data/archive_contents/ (details below). If a key 

156 matches, the corresponding value is used to compute the response. If no key 

157 matches, an response with empty content is returned. 

158 

159 If several keys match, the one that was inserted most recently in 

160 url_mapping is used. 

161 

162 Specifying keys 

163 --------------- 

164 Keys of url_mapping can be: 

165 

166 - a `str`: it is used as a glob pattern, matched against the url with 

167 fnmatch (If special characters need to be matched literally they can be 

168 escaped with []). For example: 

169 '*' matches everything 

170 '*example.com/*' matches 'https://www.example.com/data' 

171 but not 'https://example.org' 

172 

173 - a `re.Pattern` (ie a compiled regex): it is matched against the url, and 

174 groups can be used to capture parts needed to construct the response. For 

175 example: 

176 re.compile(r'.*example.org/subject_(\d+)\.tar\.gz') 

177 matches 'https://example.org/subject_12.tar.gz' and captures '12' 

178 but does not match 'https://example.org/subject_12.csv' 

179 

180 If none of the keys in url_mapping matches, the Sender turns to the 

181 contents of nilearn/datasets/tests/data/archive_contents. Files in this 

182 directory or any subdirectory are used to build responses that contain zip 

183 or tar archives containing a certain list of files. (.py and .pyc files are 

184 ignored) 

185 The first line of a file in archive_contents is a glob pattern stating to 

186 which urls it applies. If it matches, the subsequent lines are paths that 

187 will exist inside the archive. The files created in the archive are 

188 empty. For example, if a file looks like: 

189 https://example.org/subj_*.tar.gz 

190 README.txt 

191 data/img.nii.gz 

192 data/labels.csv 

193 the response will be a tar gzipped archive with this structure: 

194 . 

195 ├── data 

196 │   ├── img.nii.gz 

197 │   └── labels.csv 

198 └── README.txt 

199 

200 Moreover, if the first line starts with 'format:' it is used to determine 

201 the archive format. For example: 'format: zip', 'format: gztar' (see 

202 `shutil` documentation for available formats). In this case the second line 

203 contains the url pattern and the rest of the file lists the contents. 

204 The paths for the archive contents must use '/' as path separator, it gets 

205 converted to the OS's separator when the file is read. 

206 A helper script is provided in 

207 nilearn/datasets/tests/data/list_archive_contents.sh to generate such files 

208 from a url. 

209 

210 Finally, if no key and no file matches the request url, a response with an 

211 empty content is returned. 

212 

213 Specifying values 

214 ----------------- 

215 

216 Once a key matches, the corresponding value is used to build a response. 

217 The value can be: 

218 - a callable: it is called as value(match, request), where request is the 

219 input `Request` object, and match is the url if the key was a string and 

220 the `re.Match` resulting from matching the key if it was a `re.Pattern`. 

221 The result of this call is then processed as described below. 

222 - an instance of the Response class: it used without modification. 

223 - a `bytes`: result is a Response with status 200 and these bytes as 

224 content. 

225 - a str: if the key was a `re.Pattern`, the value can contain 

226 backreferences that are replaced with groups matched in the url, e.g. 

227 \1, \g<groupname>. The resulting string is then encoded with UTF-8 to 

228 build the response content. For example: 

229 re.compile(r'.*example\.org/(.*)'): r'hello, \1' 

230 results in b'hello, nilearn' if the url is https://example.org/nilearn 

231 - an int: results in an response with this status code. The content is 

232 b"ERROR" if the status code is in [400, 600[ and b"OK" otherwise 

233 - an `Exception`: it is raised 

234 - a `pathlib.Path`: the contents of the response are the contents of that 

235 file. (can also be anything that has a `read_bytes` attribute, 

236 e.g a `pathlib2.Path`) 

237 - an object with a `to_filename` method, eg a Nifti1Image: it is serialized 

238 to .nii.gz to produce the response content. 

239 

240 To help construct values that mock downloaded archives, this module 

241 provides `dict_to_archive` and `list_to_archive` helper functions; more 

242 details in their docstrings. 

243 

244 Inspecting history 

245 ------------------ 

246 Senders record all sent requests in `sent_requests`, the visited urls in 

247 `visited_urls`, and the number of sent requests in `url_count` 

248 

249 """ 

250 is_mock = True 

251 

252 def __init__(self): 

253 self.url_mapping = OrderedDict() 

254 self.sent_requests = [] 

255 self._archive_contents_index = _index_archive_contents()

257 @property 

258 def visited_urls(self): 

259 return [request.url for request in self.sent_requests]

266 if isinstance(request, str): 

267 request = Request(request) 

268 self.sent_requests.append(request) 

269 for key, value in list(self.url_mapping.items())[::-1]: 

270 match = self.match(key, request.url) 

271 if match is not None: 

272 return self.get_response(value, match, request) 

273 for key, file_path in self._archive_contents_index.items(): 

274 match = self.match(key, request.url) 

275 if match is not None: 

276 return Response(

284 def match(self, key, url): 

285 if isinstance(key, type(re.compile(r".*"))): 

286 return key.match(url) 

287 elif isinstance(key, str) and fnmatch.fnmatch(url, key): 

288 return url 

289 else: 

290 return None

292 def get_response(self, response, match, request): 

293 if hasattr(response, "__call__"): 

294 response = response(match, request) 

295 

296 if isinstance(response, Response): 

297 return response 

298 elif isinstance(response, Exception): 

299 raise response 

300 elif isinstance(response, int): 

301 if 400 <= response < 600: 

302 return Response(b"ERROR", request.url, status_code=response) 

303 else: 

304 return Response(b"OK", request.url, status_code=response) 

305 elif hasattr(response, "to_filename"): 

306 return Response(serialize_niimg(response), request.url) 

307 elif hasattr(response, "read_bytes"): 

308 return Response(response.read_bytes(), request.url) 

309 elif isinstance(response, str): 

310 if isinstance(match, type(re.match(r".*", ""))): 

311 response = match.expand(response) 

312 response = response.encode("utf-8") 

313 return Response(response, request.url) 

314 elif isinstance(response, bytes): 

315 return Response(response, request.url) 

316 else: 

317 raise TypeError( 

318 "Don't know how to make a Response from: {}".format(response) 

319 )

352def _add_to_archive(path, content): 

353 path.parent.mkdir(exist_ok=True, parents=True) 

354 if hasattr(content, "to_filename"): 

355 content.to_filename(str(path)) 

356 elif hasattr(content, "is_dir") and hasattr(content, "is_file"): 

357 if content.is_file(): 

358 shutil.copy(str(content), str(path)) 

359 elif content.is_dir(): 

360 shutil.copytree(str(content), str(path)) 

361 else: 

362 raise FileNotFoundError( 

363 "Not found or not a regular file " 

364 "or a directory {}".format(content) 

365 ) 

366 elif isinstance(content, str): 

367 with path.open("w") as f: 

368 f.write(content) 

369 elif isinstance(content, bytes): 

370 with path.open("wb") as f: 

371 f.write(content) 

372 else: 

373 with path.open("wb") as f: 

374 pickle.dump(content, f)

377def dict_to_archive(data, archive_format="gztar"): 

378 """Transform a {path: content} dict to an archive 

379 

380 Parameters 

381 ---------- 

382 data : dict 

383 Keys are strings or `pathlib.Path` objects and specify paths inside the 

384 archive. (If strings, must use the system path separator.) 

385 Values determine the contents of these files and can be: 

386 - an object with a `to_filename` method (e.g. a Nifti1Image): it is 

387 serialized to .nii.gz 

388 - a `pathlib.Path`: the contents are copied inside the archive (can 

389 point to a file or a directory). (can also be anything that has 

390 `is_file` and `is_directory` attributes, e.g. a `pathlib2.Path`) 

391 - a `str` or `bytes`: the contents of the file 

392 - anything else is pickled. 

393 

394 archive_format : str, optional (default="gztar") 

395 The archive format. See `shutil` documentation for available formats. 

396 

397 Returns 

398 ------- 

399 bytes : the contents of the resulting archive file, to be used for example 

400 as the contents of a mock response object (see Sender). 

401 

402 Examples 

403 -------- 

404 if `data` is `{"README.txt": "hello", Path("Data") / "labels.csv": "a,b"}`, 

405 the resulting archive has this structure: 

406 . 

407 ├── Data 

408 │   └── labels.csv 

409 └── README.txt 

410 

411 where labels.csv and README.txt contain the corresponding values in `data` 

412 

413 """ 

414 with tempfile.TemporaryDirectory() as root_tmp_dir: 

415 root_tmp_dir = Path(root_tmp_dir) 

416 tmp_dir = root_tmp_dir / "tmp" 

417 tmp_dir.mkdir() 

418 for path, content in data.items(): 

419 _add_to_archive(tmp_dir / path, content) 

420 archive_path = shutil.make_archive( 

421 str(root_tmp_dir / "archive"), archive_format, str(tmp_dir) 

422 ) 

423 with open(archive_path, "rb") as f: 

424 return f.read()
